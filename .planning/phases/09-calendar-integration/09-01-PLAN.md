# Plan 09-01: Calendar Engine Enhancements

**Phase:** 9 - Calendar Integration
**Requirements:** CAL-02 (partial), CAL-03
**File:** `src/lib/calendarEngine.ts`

## Goal

Add two new exports to the calendar engine:
1. `getCalendarPermissionStatus()` — synchronous read of cached permission status
2. `checkCalendarPermission()` — async check without triggering system dialog (uses `getCalendarPermissionsAsync` not `requestCalendarPermissionsAsync`)

These are needed by the Settings screen (CAL-02) to display current permission status, and ensure the engine properly refreshes its cache when the app regains focus.

## Tasks

### Task 1: Add `getCalendarPermissionStatus()` export
**File:** `src/lib/calendarEngine.ts`
**Action:** Add a new exported function that returns the module-level `_permissionStatus` value.

```typescript
export function getCalendarPermissionStatus(): CalendarPermissionStatus {
  return _permissionStatus;
}
```

### Task 2: Add `checkCalendarPermission()` export
**File:** `src/lib/calendarEngine.ts`
**Action:** Add an async function that checks (not requests) the current calendar permission status using `getCalendarPermissionsAsync()` and updates the module-level cache.

```typescript
export async function checkCalendarPermission(): Promise<CalendarPermissionStatus> {
  const ExpoCalendar = getExpoCalendar();
  if (!ExpoCalendar) {
    return _permissionStatus;
  }
  try {
    const { status } = await ExpoCalendar.getCalendarPermissionsAsync();
    _permissionStatus = status === "granted" ? "granted" : "denied";
  } catch {
    // Fail silently — retain existing cached value
  }
  return _permissionStatus;
}
```

## Verification
- `getCalendarPermissionStatus()` returns `"undetermined"` initially
- `checkCalendarPermission()` returns a valid status without showing system dialog
- All existing exports still work (no regressions)
