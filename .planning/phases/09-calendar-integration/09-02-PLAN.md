# Plan 09-02: Garden Walk Setup Calendar Permission Step (CAL-01)

**Phase:** 9 - Calendar Integration
**Requirements:** CAL-01
**File:** `app/garden-walk-setup.tsx`

## Goal

Add an optional, skippable calendar permission section to the Garden Walk setup screen. The user can grant calendar access or skip without consequence. The section appears below the time picker and above the confirm button.

## Design

A new section between the time picker and the "Sounds lovely" button:
- Section header: "CALENDAR" (same style as Day/Time headers)
- Explanation text: gentle, optional tone — "If you'd like, Kinship can check your calendar to suggest capturing memories after events with people in your garden."
- Two states:
  - **Not yet asked**: Show "Allow calendar access" button (sage outline) + "Skip" text link
  - **Granted**: Show green checkmark + "Calendar connected" text
  - **Denied/Skipped**: Show subtle "You can enable this later in Settings" text

## Tasks

### Task 1: Add calendar state and handlers
**Action:** Import `requestCalendarPermission` from calendarEngine. Add state for calendar step. Add handlers.

```typescript
import { requestCalendarPermission } from "@/lib/calendarEngine";
import type { CalendarPermissionStatus } from "@/lib/calendarEngine";

// In component:
const [calendarStatus, setCalendarStatus] = useState<"pending" | "granted" | "skipped">("pending");

const handleAllowCalendar = useCallback(async () => {
  const status = await requestCalendarPermission();
  setCalendarStatus(status === "granted" ? "granted" : "skipped");
}, []);

const handleSkipCalendar = useCallback(() => {
  setCalendarStatus("skipped");
}, []);
```

### Task 2: Add calendar permission UI section
**Action:** Insert new Animated.View section between the time picker and confirm button. Show different content based on `calendarStatus`.

- When "pending": Show explanation + "Allow calendar access" button + "Skip" link
- When "granted": Show checkmark + "Calendar connected" message
- When "skipped": Show "You can enable this later in Settings" message

### Task 3: Tone validation
**Action:** Ensure all copy passes tone rules:
- No "we need", "you should", "don't forget"
- Use "If you'd like", "You can", "No pressure"
- Calendar access is presented as entirely optional

## Verification
- Garden Walk setup shows calendar section
- Tapping "Allow" triggers the system permission dialog
- Granting shows "Calendar connected"
- Tapping "Skip" shows "enable later in Settings" message
- "Sounds lovely" button still works regardless of calendar choice
- Skipping has zero consequences on the rest of the app
