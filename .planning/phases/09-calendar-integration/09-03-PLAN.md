# Plan 09-03: Settings Calendar Permission Management (CAL-02)

**Phase:** 9 - Calendar Integration
**Requirements:** CAL-02
**File:** `app/settings/index.tsx`

## Goal

Add a "Calendar" row to the Settings screen that shows the current permission status and allows the user to navigate to device settings to change it.

## Design

In the "App" section of settings, add a new row between "Privacy & Data" and "Account":
- Icon: Calendar icon from lucide-react-native
- Label: "Calendar"
- Right side: status badge showing "Connected" (green) or "Not connected" (gray)
- On press: Opens device settings via `Linking.openSettings()`

## Tasks

### Task 1: Import dependencies
**Action:** Import `Calendar` icon from lucide, `Linking` from react-native, `useFocusEffect` from expo-router, `checkCalendarPermission` and `CalendarPermissionStatus` from calendarEngine.

### Task 2: Add permission status state and refresh
**Action:** Add state for calendar status. Use `useFocusEffect` to re-check permission status each time Settings screen gains focus (handles user returning from device settings).

```typescript
const [calendarStatus, setCalendarStatus] = useState<CalendarPermissionStatus>("undetermined");

useFocusEffect(useCallback(() => {
  (async () => {
    const status = await checkCalendarPermission();
    setCalendarStatus(status);
  })();
}, []));
```

### Task 3: Enhance SettingsRow component
**Action:** Add optional `rightElement` prop to SettingsRow so we can show a status badge instead of the default chevron.

### Task 4: Add Calendar row to App section
**Action:** Insert Calendar row in the App section. Show status badge on the right. On press, call `Linking.openSettings()`.

```typescript
<SettingsRow
  icon={Calendar}
  label="Calendar"
  onPress={() => Linking.openSettings()}
  rightElement={
    <View style={{ flexDirection: "row", alignItems: "center", gap: 6 }}>
      <View style={{
        width: 8, height: 8, borderRadius: 4,
        backgroundColor: calendarStatus === "granted" ? "#6B9E6B" : colors.warmGray,
      }} />
      <Text style={{
        fontFamily: fonts.sans, fontSize: 13,
        color: calendarStatus === "granted" ? "#6B9E6B" : colors.warmGray,
      }}>
        {calendarStatus === "granted" ? "Connected" : "Not connected"}
      </Text>
    </View>
  }
/>
```

## Verification
- Settings screen shows "Calendar" row in App section
- Row shows "Connected" with green dot when calendar permission is granted
- Row shows "Not connected" with gray dot when permission is denied/undetermined
- Tapping the row opens device settings
- Returning to Settings refreshes the status (via useFocusEffect)
- No crash if expo-calendar module is unavailable (Expo Go)
