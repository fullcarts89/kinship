# Phase 8: Garden Walk - Research

**Researched:** 2026-02-27
**Domain:** Garden Walk screen refactoring + suggestion engine integration
**Confidence:** HIGH

## Summary

The Garden Walk screen (`app/garden-walk.tsx`) currently has three issues that need addressing for WALK-01 and WALK-02:

1. **"Capture a moment" / "Anything on your mind" section** (lines 451-503) is a standalone gold-tinted card at the bottom of the screen that links to `/memory/add`. The empty state (lines 199-249) also contains a "Capture a moment" prompt. Both must be removed per WALK-01.

2. **The screen already uses `generateSuggestions()`** from the suggestion engine (line 99), which is good. However, it calls the engine directly instead of using the `useSuggestions` hook (which adds calendar match integration). The `getContextLine()` helper (lines 45-63) provides the per-person context reason, but its copy is minimal and doesn't leverage the rich `reason` field already generated by the suggestion engine (INTL-05 personalized copy).

3. **The empty state needs rework.** The current empty state says "Your garden is just getting started. Capture a moment to help it grow." — this is a capture-focused prompt. It needs to be replaced with a tone-appropriate empty state about no suggested people to tend.

**Primary recommendation:** Remove the capture prompt card and the capture-focused empty state. Switch from direct `generateSuggestions()` call to the `useSuggestions` hook (which includes calendar matches). Use the suggestion engine's `reason` field for context lines instead of the minimal `getContextLine()` helper.

## Architecture Patterns

### Current Garden Walk Data Flow
```
usePersons() + useMemories() + useAllInteractions()
  → generateSuggestions(persons, memories, interactions, undefined, 5)
  → split into memoryResurface + plantsToTend
  → render cards
```

### Target Data Flow
```
usePersons() + useMemories() + useAllInteractions()
  → useSuggestions(persons, memories, interactions, 5)
    → internally fetches calendar matches
    → calls generateSuggestions(persons, memories, interactions, calendarMatches, 5)
  → split into memoryResurface + plantsToTend
  → render with engine's `reason` field
```

### Key Files
- `app/garden-walk.tsx` — The screen to modify (530 lines)
- `src/lib/suggestionEngine.ts` — Engine with 24h recency exclusion, personalized copy, 7-day memory lag (all Phase 7 work)
- `src/hooks/useSuggestions.ts` — Hook wrapper that adds calendar matches

### Sections to Remove (WALK-01)
1. **Capture Prompt Card** (lines 451-503): Gold-tinted card with "Anything on your mind today?" + "Capture a moment" link
2. **Empty State capture link** (lines 199-249): "Capture a moment" button within the empty state card

### Context Line Improvement (WALK-02)
The current `getContextLine()` returns minimal, generic text:
- `drift_reconnect` → "Well-rooted"
- `post_event_capture` → "Might enjoy capturing a moment"
- `general_reach_out` → "Might enjoy a note"

The suggestion engine's `reason` field already contains rich, personalized copy (INTL-05):
- Drift: "Your friendship with Sarah has deep roots. A small moment could be meaningful."
- General: "A gentle thought for Dad could be a nice way to start."

We should display the `reason` field (truncated if needed) instead of the generic `getContextLine()`.

## Common Pitfalls

### Pitfall 1: Removing capture prompt but leaving empty state capture link
**What goes wrong:** Empty state still shows "Capture a moment" link
**How to avoid:** Remove both: the gold card AND the capture link in the empty state

### Pitfall 2: useSuggestions hook signature mismatch
**What goes wrong:** The hook takes (persons, memories, interactions, limit) — the current screen passes data differently
**How to avoid:** Replace the direct `generateSuggestions()` call and related `useMemo` with a single `useSuggestions()` call

### Pitfall 3: Hooks before early returns
**What goes wrong:** React hooks must be called unconditionally before any early returns
**How to avoid:** `useSuggestions` must be called at the top level of the component, alongside other hooks
