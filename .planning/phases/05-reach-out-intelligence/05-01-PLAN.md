# Plan 05-01: Memory Carousel for Reach-Out Screen

**Phase:** 5 - Reach-Out Intelligence
**Wave:** 1 (no dependencies on other Phase 5 plans)
**Requirements:** REACH-01

## Goal

Add a horizontally swipeable memory carousel to the reach-out flow that shows the person's recent memories (past year, most recent first) with content and emotion tags, giving the user context before choosing a reach-out action. The carousel is skippable — a person with no memories reaches the action buttons without error.

## Tasks

### Task 1: Create MemoryCarousel component

**File:** `src/components/MemoryCarousel.tsx` (NEW)

**Actions:**
1. Create a new component `MemoryCarousel` that accepts:
   - `memories: Memory[]` — already filtered and sorted
   - `style?: ViewStyle` — optional container style
2. Render a horizontal `FlatList` with:
   - `horizontal={true}`
   - `pagingEnabled={true}`
   - `showsHorizontalScrollIndicator={false}`
   - `nestedScrollEnabled={true}` (prevents gesture conflict with tab navigation)
   - `snapToInterval={CARD_WIDTH + CARD_GAP}` for consistent snapping
   - `decelerationRate="fast"` for snappy feel
   - `contentContainerStyle` with horizontal padding for edge-card centering
3. Each card renders:
   - Memory content text (truncated to 3 lines with `numberOfLines={3}`)
   - Emotion tag chip (if present) using `emotionEmojis` from `@/lib/formatters` and `formatEmotionLabel`
   - Photo thumbnail (if `photo_url` exists) as card background with overlay
   - Relative date using `formatRelativeDate` from `@/lib/formatters`
   - Card styled with `LinearGradient` background (sagePale to sageLight), rounded corners, shadow
4. Add dot indicators below the carousel showing current page position
5. Use design tokens (sage, sagePale, sageLight, cream, warmGray, nearBlack, fonts)

**Verification:** Component renders with mock memory data; swipe advances cards; emotion chip visible; empty array renders nothing (parent handles empty state).

### Task 2: Integrate carousel into reach-out BridgeScreen

**File:** `app/reach-out/[id].tsx` (MODIFY)

**Actions:**
1. Import `MemoryCarousel` from `@/components/MemoryCarousel`
2. Filter `memories` from `usePersonMemories` to past year:
   ```typescript
   const oneYearAgo = new Date();
   oneYearAgo.setFullYear(oneYearAgo.getFullYear() - 1);
   const recentMemories = memories
     .filter(m => new Date(m.created_at) >= oneYearAgo)
     .sort((a, b) => new Date(b.created_at).getTime() - new Date(a.created_at).getTime());
   ```
3. Replace the single-memory preview section (the `{recentMemory && (...)}` block and `{!recentMemory && (...)}` fallback) with:
   - If `recentMemories.length > 0`: render `<MemoryCarousel memories={recentMemories} />`
   - If `recentMemories.length === 0`: render the existing gentle fallback text ("A small hello can mean a lot")
4. Remove the `getBestMemoryForReachOut` and `getMemoryContextLabel` imports (no longer needed — carousel replaces single-memory selection)
5. Keep the PlantBridgeIllustration, primary text, suggested openings, and action buttons unchanged for now (Wave 2 handles action contextuality)

**Verification:** Opening reach-out for a person with memories shows swipeable carousel; for a person with no memories shows fallback text; carousel swipe does not conflict with tab navigation; memory cards show content + emotion tags.
