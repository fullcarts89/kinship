  <action>
Add a `useMemory(id: string)` hook at the end of `src/hooks/useMemories.ts`. The hook:

1. Takes a single `id: string` parameter.
2. Uses `useState<Memory | null>(null)` and `useState(true)` for isLoading.
3. Uses a `useCallback` wrapping the lookup logic, with `[id]` dependency array.
4. Inside the callback: sets isLoading true, wraps in try/catch. The try block attempts `throw new Error("mock mode")` (placeholder for future Supabase `memoryService.getMemoryById(id)` call). The catch block searches `[...locallyCreatedMemories, ...mockMemories].find((m) => m.id === id) ?? null` and sets the result. Finally block sets isLoading false.
5. A `useEffect` calls the fetch function on mount / id change.
6. Returns `{ memory, isLoading }`.

This follows the exact same pattern as `usePersonMemories` — try Supabase, catch falls back to mock data search. The only difference is `.find()` instead of `.filter()`.
