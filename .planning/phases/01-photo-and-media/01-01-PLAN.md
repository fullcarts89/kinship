**Changes:**

2. Rewrite `useMemories()` to merge mock + local data when Supabase fetch fails:
   - In the catch block, instead of setting memories to empty array, set to `[...locallyCreatedMemories, ...mockMemories]` (locally created first, then mock, for correct ordering by recency).
   - This ensures the home screen and any "all memories" consumer sees both mock and newly created memories.

3. Rewrite `usePersonMemories(personId)` similarly:
   - In the catch block, merge and filter: `[...locallyCreatedMemories, ...mockMemories].filter(m => m.person_id === personId)`.
   - This ensures person profile shows both mock and newly created memories for that person.

4. Rewrite `useCreateMemory()` to add mock fallback in the catch block:
   ```typescript
   export function useCreateMemory() {
     const [isCreating, setIsCreating] = useState(false);
     const [error, setError] = useState<Error | null>(null);

     const createMemory = useCallback(
       async (memory: Omit<MemoryInsert, "user_id">): Promise<Memory> => {
         try {
           setIsCreating(true);
           setError(null);
           const created = await memoryService.createMemory(memory);
           return created;
         } catch {
           // Mock mode — Supabase not configured
           const newMemory: Memory = {
             id: `m-local-${Date.now()}`,
             user_id: "u1",
             person_id: memory.person_id,
             content: memory.content,
             emotion: memory.emotion ?? null,
             photo_url: memory.photo_url ?? null,
             created_at: new Date().toISOString(),
           };
           locallyCreatedMemories.unshift(newMemory);
           return newMemory;
         } finally {
           setIsCreating(false);
         }
       },
       []
     );

     return { createMemory, isCreating, error };
   }
   ```

**Key details:**
- Use `unshift` (not push) so newest memories appear first in the array.
- The mock fallback does NOT set error state — it silently succeeds, matching the user expectation that saving works.
- The catch block no longer re-throws — it returns the locally created memory instead.
- `photo_url: memory.photo_url ?? null` ensures the photo URI is stored.

**Important:** Keep the existing JSDoc comment at the top of the file but update it to reflect that the file now falls back to mock data when Supabase is unavailable.

To:

  mediaTypes: ["images"],

const created = await createMemory({

**Why:** `photoUri` is already in scope (captured by the picker and stored in component state) but was never passed to `createMemory`. Adding `photo_url: photoUri ?? null` ensures the URI reaches the mock persistence layer (or Supabase when connected).
