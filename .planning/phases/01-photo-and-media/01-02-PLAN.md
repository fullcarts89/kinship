**Key details:**

- `width: "100%"` fills the card within its `overflow: "hidden"` + `borderRadius: 20` container.
- `resizeMode="cover"` fills without distortion.

**Do NOT change** any other part of MemorySpotlight — the fade animation, the content area, the person name, the date, and the onPress navigation should all remain as-is.

**Part B — app.json plugin registration:**

In `app.json`, add `expo-image-picker` to the `plugins` array with a permission description that matches the app's garden tone. Add it after the existing `"expo-asset"` entry (before the expo-contacts entry):

```json
[
  "expo-image-picker",
  {
    "photosPermission": "Allow Kinship to access your photos to attach them to memories in your garden."
  }
],
```

**Why:** The `expo-image-picker` plugin adds `NSPhotoLibraryUsageDescription` to Info.plist on iOS and the corresponding Android permission. Without it, production builds (TestFlight, App Store) will not request photo library access correctly. Expo Go bundles this by default, so the absence only causes issues in production.
